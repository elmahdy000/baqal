# Baqal — Multi-Tenant SaaS PWA Grocery Ordering Platform

## Project Name

**Baqal / بقالك**

## Project Type

Build a production-ready **multi-tenant SaaS PWA** for small grocery stores located under residential buildings, compounds, and local neighborhoods.

The platform allows the system admin to create grocery store accounts. Each grocery store can manage its products, images, prices, quantities, QR codes, orders, customers, and delivery operations.

Customers scan a QR code placed inside buildings, elevators, entrances, or WhatsApp groups. The QR opens the store’s PWA directly, allowing residents to order groceries from the store under their building without downloading a native mobile app.

---

# Core Concept

The product is not just an online grocery app.

It is a complete operating system for the local neighborhood grocery store.

The core value:

> The grocery store under the building becomes available online through a QR code.

The customer does not search for stores.
The customer scans the QR code of the building and lands directly on the assigned grocery store.

# Multi-Tenant Architecture

Each grocery store is a tenant.

Every store-owned record must include:

```txt
storeId
```

The system must prevent data leakage between stores.

A store owner should only access data belonging to his own store.

Super admin can access all stores.

Customer routes should resolve the correct store based on:

```txt
QR code
building code
store slug
store id
```

Example customer URL:

```txt
https://baqal.app/b/BUILDING_CODE
https://baqal.app/store/store-slug
```

# Critical Business Logic

## New Order Logic

When customer places order:

1. Validate store is active
2. Validate store is open
3. Validate products belong to same store
4. Validate stock availability
5. Create order
6. Create order items with price snapshot
7. Reserve or decrease stock
8. Create order status history
9. Create notification for store
10. Emit realtime event to store dashboard
11. Show tracking page to customer

---

## Order Acceptance Logic

When store accepts order:

1. Change status to ACCEPTED
2. Add status history
3. Notify customer
4. Emit realtime event
5. Keep stock deducted

---

## Order Cancellation Logic

When order is cancelled:

1. Change status to CANCELLED
2. Restore stock if needed
3. Add status history
4. Notify customer and store
5. Emit realtime event

---

## Low Stock Logic

After stock changes:

1. Check product quantity
2. If quantity <= lowStockThreshold, create low-stock notification
3. If quantity = 0, mark product out of stock
4. Emit realtime inventory alert to store dashboard

* Use TypeScript strictly
* Use clean components
* Use server-side validation
* Avoid duplicate code
* Use reusable UI components
* Use clear file naming
* Use Prisma relations properly
* Add comments only where needed
* Keep business logic in service layer

# Final Goal

The final product should allow this real-world flow:

1. Super admin creates a grocery store account.
2. Store owner logs in.
3. Store owner adds products, prices, images, and quantities.
4. Admin or store owner creates QR code for a building.
5. QR is printed and placed inside the building.
6. Resident scans QR.
7. Resident opens the PWA directly.
8. Resident selects products and places an order.
9. Store receives realtime alert.
10. Store accepts and prepares the order.
11. Customer tracks order status.
12. Delivery sends order to apartment.
13. Inventory updates automatically.
14. Admin monitors store activity and subscriptions.

Build this as a serious SaaS product, not a demo.
